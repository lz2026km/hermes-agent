## G系列部署规范（重要！）

### 核心原则
G003~G008所有SPA系统必须用 `vite preview`，禁止用 `http-server`（不支持React Router SPA fallback）

### 标准部署流程

**第一步：本地构建**
```bash
cd /home/admin/hermes-agent/web/G00X
npm run build    # 生成 dist/
```

**第二步：打包传输**
```bash
tar -czf dist.tar.gz dist/
scp -o StrictHostKeyChecking=no dist.tar.gz admin@8.137.71.217:/tmp/
```

**第三步：远程部署（关键！）**
```bash
ssh admin@8.137.71.217
# 1. 解压dist
tar -xzf /tmp/dist.tar.gz

# 2. 杀掉旧进程
pkill -f "vite preview.*PORT" 2>/dev/null

# 3. 启动vite preview（关键：用 nohup + exit 组合）
cd /home/admin/hermes-agent/web/G00X
nohup npx vite preview --port PORT --host 0.0.0.0 > /tmp/g00x.log 2>&1 &
exit
```

**为什么要 `exit`？**
- SSH默认等待前台进程，即使 `nohup &` 也可能被SSH挂起
- `exit` 强制关闭SSH会话，让后台进程真正脱离SSH会话运行
- 这是在 `nohup ... &` 之后立即执行的，不是等进程启动

**第四步：验证**
```bash
curl -s -o /dev/null -w "HTTP:%{http_code}" http://localhost:PORT/
curl -s -o /dev/null -w "HTTP:%{http_code}" http://localhost:PORT/g00x/具体页面
```

### 常见错误
1. **SSH超时**：命令里必须有 `exit`，否则SSH等待进程导致超时
2. **进程立即退出**：检查日志 `cat /tmp/g00x.log`
3. **端口冲突**：`pkill` 要在启动前执行
4. **路径错误**：G007实际路径是 `/home/admin/hermes-agent/web/G007`（不是 `/home/admin/g007`）

### 验证命令速查
