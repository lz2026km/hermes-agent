## G005白屏修复（下午半天）

### 问题描述
G005放射科RIS系统部署到5179端口后，首页正常但SPA路由(/worklist等)白屏404

### 根因
用了 `http-server` 部署——这是纯静态文件服务器，**不支持React Router SPA fallback**

### 教训
- G006/G003能正常跑SPA是因为用 `vite dev` 开发服务器
- http-server在G002/G005等项目一直作为"能用"的备选，但实际上SPA场景下是有缺陷的
- **没有先对比其他系统的部署方式**，就默认http-server是正确答案
- 反思不彻底：之前"对比G006"时没有深究"为什么G006能跑SPA"

### 修复方案
```bash
# 错误方式
npx http-server . -p 5179  # ❌ 不支持SPA路由

# 正确方式
cd /home/admin/hermes-agent/web/G005
npx vite preview --port 5179  # ✅ vite天然支持React Router fallback
```

### 修复后验收

|------|------|
| / (首页) | element_count: 65 ✅ |
| /worklist | element_count: 2888 ✅ |
| /patient | 200 ✅ |
| Console errors | 0 ✅ |

## 记住的教训

**以后部署任何G系列SPA系统，必须用 `vite dev` 或 `vite preview`，不能用 http-server**

http-server只适合纯静态页面（没有React Router子路由的场景）
